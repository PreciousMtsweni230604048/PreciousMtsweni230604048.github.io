Reflection on Coding in Markdown

Situation

Before this project, I had never written a single line of Markdown. I had heard of GitHub before, mostly in passing during lectures, but I had never actually sat down and used it for anything meaningful. When I saw that the portfolio had to be built using Markdown, I won't lie — I was a little nervous.

Task

I needed to code my entire CV and portfolio content in Markdown and have it display properly on GitHub. It had to look clean and professional, not like something thrown together at the last minute.

Action

I started small. I watched a few tutorials, opened a README file, and just started experimenting. I broke things, fixed them, broke them again. I learned how headings work, how to create tables for my CV, how to add links and format text. It was frustrating at first but the more I practised, the more it started to click.

Result

By the end of it, I could write Markdown without constantly needing to look things up. My CV came out looking structured and easy to read. More than that, I now understand why developers use it — it is simple, clean, and gets the job done without unnecessary complexity.

Reflection on Mock Interview Video Experience

Situation

I have always found interviews uncomfortable. The idea of sitting in front of a camera and pretending to be in a job interview felt awkward before I even started. But I knew it was something I needed to take seriously because one day it would not be practice anymore.

Task

I had to record a mock interview, embed it into my portfolio, and reflect on how it went. The point was not just to film something and move on, but to actually think about how I came across and what I could do better.

Action

I wrote down possible interview questions and thought carefully about how to answer them using real examples from my studies and personal experience. I filmed it in a quiet room, made sure the lighting was decent, and did a few takes before I was satisfied. Uploading and embedding it took some trial and error but I figured it out.

Result

Watching myself back was uncomfortable but useful. I noticed I spoke too fast when I was nervous and I sometimes gave vague answers when I should have been more specific. That awareness is something I can actually work on. I feel more prepared for a real interview now than I did before this exercise.

Reflection on the Use of GitHub Pages

Situation

Getting content onto GitHub was one thing. Getting it to actually live on the internet as a real website was something else entirely. I had never deployed anything before and I was not sure what to expect from the process.

Task

I needed to publish my portfolio using GitHub Pages and make sure everything worked correctly on the live site — the formatting, the links, the embedded video, all of it.

Action

I went into the repository settings and enabled GitHub Pages. Then I spent time going through the live site carefully, checking that nothing had broken in the transition from the editor to the browser. A few things needed fixing and I went back and corrected them until the site looked the way I wanted it to.

Result

Seeing my portfolio go live with its own URL was genuinely satisfying. It made everything feel real — like this was not just a school submission but something I could actually show to someone. That feeling made the whole process worth it and it gave me confidence that I can build and publish things on my own.
